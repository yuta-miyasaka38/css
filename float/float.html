(回避法１) 親要素もfloatする

親要素
float: left;
floatしない
※親要素と子要素の次元が同じになる。(浮く)
　→ 親要素の中に子要素が入る

(回避法２) 親要素にoverflowを適用 ~hidden~

親要素
float: left;
floatしない
※overflowは「visible以外は中の要素の高さを加味する」性質を使用
hiddenの場合
※「あえてはみ出したい要素がある場合に使えない」

(回避法２) 親要素にoverflowを適用 ~scroll.(auto)~

親要素
float: left;
floatしない
※overflowは「visible以外は中の要素の高さを加味する」性質を使用
scrollの場合 ※hiddenかautoが一般的

(回避法３) 疑似要素を使う～clearfix～

親要素
float: left;
floatしない
    ※親要素へのfloatや、overflowと違い、レイアウト崩れが発生しない
    ※クラスclearfixを、floatを使用している親要素につける
    ※:after　contentプロパティと一緒に使うことで、ボックス内の末尾に新たなインラインボックスをつくる
    ※:afterにより生成したボックスに clear:both;　を適用する
    ※インライン要素をブロック要素にする　display: block;

    .clearfix:after {
        content: "";         /* 擬似要素を実体化 */
        clear: both;        /* floatを解除する */
        display: block;     /* ブロック要素にする */
    }


<結論>

float解除の優先順位
(1)親要素にclearfix (class)
(2)親要素にoverflow (プロパティ)
(3)親要素にfloat　(プロパティ)
